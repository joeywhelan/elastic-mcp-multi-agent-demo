Elastic Agent Builder

Elastic has a full agentic AI framework built into Kibana.

  • GUI-based agent creation and management
  • ES|QL-powered tool definitions
  • Built-in MCP server endpoint
  • Hybrid search, semantic fields, reranking — all accessible as tools

But the real power is that these tools are not locked into our framework.

MCP: The Bridge

The Model Context Protocol is an open standard for connecting AI agents to data sources and tools.

How It Works

  • Elastic exposes tools via a standard MCP endpoint
  • Any MCP-compatible client connects over HTTP
  • Tools are discovered and invoked at runtime
  • Framework-agnostic by design

What Elastic Brings to Your Agents

Search Capabilities

  • Hybrid search — lexical + semantic in one query
  • FUSE — Linear or RRF fusion of multiple search strategies
  • RERANK — neural reranking (e.g. Jina Reranker v3)
  • semantic_text — automatic embeddings at index time

Data Platform

  • ES|QL — powerful piped query language
  • Structured + unstructured data in one platform
  • Serverless — zero infrastructure management
  • Scales to billions of documents

All of this is exposed as MCP tools that any agent can call.

ES|QL-Powered MCP Tools

Define once in Elastic. Use from any framework.

Product Search

FROM products METADATA _score
| FORK
  (WHERE MATCH(name, ?search_term,
    {"boost": 2.0})
    OR MATCH(description, ?search_term)
   | SORT _score DESC | LIMIT 100)
  (WHERE MATCH(description.embedding,
    ?search_term)
   | SORT _score DESC | LIMIT 100)
| FUSE LINEAR
    WITH { "normalizer": "minmax" }
| RERANK ?search_term ON description
    WITH { "inference_id":
           ".jina-reranker-v3" }
| SORT _score DESC | LIMIT 3

Hybrid search + fusion + reranking

Review Lookup

FROM reviews
| WHERE product_id == ?id
| KEEP review
| LIMIT 5

Simple chained lookup by product ID

Agents chain these tools: search for a product, then fetch its reviews — multi-step reasoning powered by Elastic data.
